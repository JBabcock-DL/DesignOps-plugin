## Step 10 — Present design system plan and get approval

Before writing anything to Figma or the filesystem, present a **summary card** and call **AskUserQuestion**. The full token table is **only** emitted if the designer explicitly asks for it — the default `yes` path skips straight to Step 11. This keeps approval fast and avoids dumping hundreds of rows into chat for designers who just want to push.

### 10b — Ask for approval (branch on reply)

Call **AskUserQuestion**:

> "Does this design system look correct? Reply **yes** to push all five variable collections (Plugin API creation + REST `codeSyntax` patches — 5 sequential passes, ~5–45s each depending on collection), **details** to see the full token table first, or describe any changes (e.g. 'change primary to #E63946', 'use Geist for body font', 'base radius 8px')."

Branch on the reply:

- **`yes`** (or equivalent affirmative) → proceed directly to Step 11. Do **not** emit the full table.
- **`details`** → emit the full token table in §10c below, then immediately re-ask the same AskUserQuestion. Do not loop on `details` (treat a second `details` as `yes`).
- **Change request** → route per the change-routing rules at the end of this step (§10d), re-display the summary card (and, if the designer previously asked for `details`, the updated full table), then call **AskUserQuestion** again.

### 10d — Change routing (on a change request)

When the designer requests changes instead of replying `yes` or `details`, identify which inputs need to change and loop back to the appropriate step, then **re-display the summary card** (§10a) and call **AskUserQuestion** again:

- Color changes → recompute the affected ramp(s) in Step 5, then re-derive all Theme aliases that reference that ramp in Step 6.
- Font changes → update **`typeface/display`** and **`typeface/body`** in Step 5 (and any slot metrics in Step 7 if sizes/weights should change).
- Spacing or radius changes → recompute the affected scale in Step 5 and layout aliases in Step 8.

If the designer had previously asked for `details`, also re-emit the updated full table (§10c) when you re-display the summary.

Do not proceed to Step 11 until the designer has explicitly replied **yes** (or an equivalent affirmative). **Do not** bundle the optional `tokens.css` write into this approval — that is a separate opt-in in Step 12.5 after the Figma push succeeds.
